# Core Beliefs

이 프로젝트의 운영 원칙입니다. 모든 에이전트와 기여자는 이 원칙을 따릅니다.

## 1. 로컬 우선, 외부 의존 최소화

- 모든 데이터는 로컬 파일시스템에 저장됩니다 (`~/.gemini/antigravity/.token-monitor/`).
- 외부 네트워크 요청은 LiteLLM 가격표 다운로드만 허용합니다.
- Antigravity RPC는 `127.0.0.1`에만 연결합니다.

## 2. 비공식 도구임을 인지

- Antigravity 내부 RPC 엔드포인트는 **비공식**이며 언제든 변경될 수 있습니다.
- 따라서 RPC 실패 시 항상 graceful fallback (예: estimated 모드)을 제공해야 합니다.
- RPC 스키마 변경에 대비해 파싱 로직은 방어적으로 작성합니다.

## 3. 다중 인스턴스 안전성

- 여러 VS Code 창이 동시에 실행될 수 있으므로, 파일 기반 락(`PollLock`)으로 중복 처리를 방지합니다.
- 디스크 상태(`monitor-state.json`)에서 최신 결과를 재로드하여 인스턴스 간 결과를 공유합니다.

## 4. 명시적 > 암묵적

- 토큰 모드(`reported` vs `estimated`)와 데이터 소스(`rpc-artifact` vs `filesystem`)를 항상 명시합니다.
- 추정 값은 반드시 `estimated` 라벨로 표시합니다.
- 비용 추정치는 참고용이며 재무적 판단에 사용하지 않도록 경고합니다.

## 5. 점진적 향상

- RPC 없이도 파일시스템 기반 추정이 동작합니다.
- RPC 내보내기가 활성화되면 정확한 `reported` 모드로 자동 전환됩니다.
- 새로운 기능은 기존 기능을 깨뜨리지 않고 추가됩니다.
